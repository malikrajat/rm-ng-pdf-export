# Roadmap

This roadmap outlines the planned direction and future enhancements for  
`@codewithrajat/rm-ng-pdf-export`.

The roadmap is indicative and may evolve based on user feedback, Angular ecosystem changes, and browser capabilities.

---

## Guiding Principles

The roadmap is driven by the following principles:

- Alignment with Angular’s official release and support strategy
- Stability over experimental features
- Performance and reliability across browsers
- Minimal configuration and predictable behavior
- Backward compatibility wherever feasible

---

## Short-Term Goals (Next 1–2 Releases)

### Stability and Quality
- Improve error handling and logging during PDF generation
- Enhance TypeScript typings and API consistency
- Strengthen unit and integration test coverage
- Address reported bugs and edge cases

### Documentation Enhancements
- Add advanced usage examples
- Improve troubleshooting documentation
- Provide real-world export scenarios
- Clarify CORS, font, and image handling guidelines

---

## Mid-Term Goals (3–6 Months)

### Feature Enhancements
- Configurable page headers and footers
- Improved support for custom fonts and font embedding
- Better handling of large DOM trees and long documents
- Optional page break controls via configuration

### Performance Improvements
- Reduced memory footprint during export
- Optimized rendering pipeline for large PDFs
- Better handling of high-resolution images

### Framework Alignment
- Full support for latest Angular stable releases
- Improved compatibility with standalone components
- Optimized behavior with zoneless Angular setups

---

## Long-Term Goals (6–12 Months)

### Advanced Capabilities
- Partial or incremental PDF rendering
- Improved layout consistency across browsers
- Optional accessibility tagging in generated PDFs
- Support for advanced layout scenarios

### Developer Experience
- CLI-based helpers for testing and validation
- Better integration examples with Nx and monorepos
- Enhanced debugging and development-time warnings

### Ecosystem Integration
- Clear compatibility with popular UI frameworks
- Guidance for micro frontend and module federation setups
- Extended configuration hooks for advanced use cases

---

## Out of Scope (Currently Not Planned)

The following items are not currently planned:

- Server-side PDF generation
- Frameworks other than Angular
- Heavy visual editors or WYSIWYG builders

These decisions may be revisited based on demand and feasibility.

---

## Community Feedback

Community input strongly influences the roadmap.  
Feature requests, bug reports, and improvement suggestions are welcome via:

- GitHub issues
- Pull requests
- Discussions and feedback threads

High-quality contributions and well-defined use cases are prioritized.

---

## Roadmap Disclaimer

This roadmap is provided for transparency and planning purposes only.  
Timelines and features may change without prior notice based on technical constraints or strategic direction.

---

## Summary

The focus of `rm-ng-pdf-export` is to remain:

- Stable
- Predictable
- Aligned with Angular standards
- Easy to integrate and maintain

The roadmap reflects a balance between new features, performance, and long-term maintainability.
